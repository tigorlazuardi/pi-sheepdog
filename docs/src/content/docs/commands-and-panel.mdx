# Commands and panel

## Slash commands

### `/sheepdog`

Primary operator entry point.

In interactive TUI mode it currently:

- opens the Sheepdog panel,
- lists tracked wakes,
- stays read-only in Ticket 1.

Outside interactive TUI it:

- returns read-only text status,
- includes the config path,
- does not expose write operations.

### `/sheepdog config`

Configuration helper.

In Ticket 1 it prints the Sheepdog config path under the dedicated agent dir.

## Panel contract

Ticket 1 keeps the panel read-only.

It should:

- show tracked scopes,
- let the operator inspect pending wakes,
- close cleanly without exposing CRUD actions.

## Not part of current contract

The following legacy commands are not current and should not appear in user-facing guidance:

- `/rate-limit-wakeup`
- `/rate-limit-wakeup-set`
- `/rate-limit-wakeup-clear`
- `/rate-limit`
