# Commands and panel

## Slash commands

### `/sheepdog`

Primary operator entry point.

In interactive TUI mode it should:

- open the Sheepdog panel,
- list all wakes,
- show relevance markers,
- allow add, edit, clear selected, and clear all.

Outside interactive TUI it should:

- return read-only text status,
- include the config path,
- not expose write operations.

### `/sheepdog config`

Configuration helper.

It should:

- create a minimal config file if missing,
- validate mapper rules,
- print warnings for bad rules,
- print example config guidance in command output.

## Panel contract

The panel is the only write UI in the current plan.

### Required panel actions

- Add wake
- Edit wake
- Clear selected wake
- Clear all wakes

### Add/edit fields

- Scope glob
- Relative duration

### Required add/edit behavior

- Default add scope to the current model scope when possible.
- Preview local wake time before save.
- Manual entries stay sticky until they fire, expire, or are cleared.

### Required confirmations

- Clear selected: confirm
- Clear all: stronger confirm

## Not part of current contract

The following legacy commands are not current and should not appear in user-facing guidance:

- `/rate-limit-wakeup`
- `/rate-limit-wakeup-set`
- `/rate-limit-wakeup-clear`
- `/rate-limit`
