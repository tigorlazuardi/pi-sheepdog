# Quickstart

This is the shortest blackbox path to understand Sheepdog.

## 1. Install the package

```bash
pi install git:github.com/tigorlazuardi/pi-sheepdog
```

## 2. Create the config file

Run:

```text
/sheepdog config
```

Expected result:

- Creates `${PI_CODING_AGENT_DIR:-$HOME/.pi/agent}/sheepdog/config.json` if missing.
- Prints the config path.
- Prints warnings for invalid mapper rules if present.

## 3. Trigger or simulate a cooldown

Use any provider/model path that can produce a cooldown signal. The current contract expects Sheepdog to accept:

- a provider 429 header path, or
- a final error text path, or
- a manual panel-created wake.

## 4. Inspect status

Run:

```text
/sheepdog
```

Expected result in TUI mode:

- Opens the Sheepdog panel.
- Shows pending scopes.
- Marks relevance for matching, catch-all, or other scopes.

Expected result outside TUI mode:

- Returns read-only text status.
- Includes the config path.
- Does not offer text CRUD commands.

## 5. Wait for wake behavior

When the cooldown passes:

- matching scope or `*` may follow up in the active session,
- mismatched scope stays pending and only gets a short human notice,
- startup-overdue entries expire silently.
