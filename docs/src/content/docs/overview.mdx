# Overview

**Sheepdog** is a Pi extension for **cooldown recovery only**.

It watches for provider cooldown signals, records scoped wake entries, and nudges the active Pi session when the cooldown has passed and the current main model is relevant.

## What Sheepdog does

- Detects cooldowns from provider HTTP 429 headers.
- Falls back to final agent error text when headers are not enough.
- Lets a human inspect tracked wakes from `/sheepdog`.
- Stores wake state globally under the Pi agent directory so wakes survive reloads and restarts.
- Shows a `sheepdog` footer/status signal for relevant pending wakes.

## What Sheepdog does not do

- General watchdog or crash recovery.
- Job supervision.
- Provider body JSON parsing in v1.
- Automatic model switching.
- Public telemetry export.
