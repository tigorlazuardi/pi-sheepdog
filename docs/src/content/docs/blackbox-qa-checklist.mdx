# Blackbox QA checklist

Use this page as the human acceptance checklist.

## Install and docs

- [ ] `pi install git:github.com/tigorlazuardi/pi-sheepdog` succeeds.
- [ ] Docs site builds with `cd docs && npm install && npm run build`.
- [ ] Sidebar order matches the intended reading flow.

## Command surface

- [ ] `/sheepdog` exists.
- [ ] `/sheepdog config` exists.
- [ ] Legacy `/rate-limit-wakeup*` commands are not the documented path.

## Config creation

- [ ] `/sheepdog config` creates `${agentDir}/sheepdog/config.json` when missing.
- [ ] Generated file contains only `{ "mappers": [] }`.
- [ ] Invalid rules warn without disabling later valid rules.

## Wake scenarios

### Matching 429 header path

- [ ] Cause or simulate a provider 429 with retry/reset headers.
- [ ] Sheepdog records a pending wake for the relevant scope.
- [ ] Footer shows a matching timer for the current model scope.

### Error-text fallback path

- [ ] Cause or simulate a final agent error text containing a rate-limit indicator and parseable cooldown duration.
- [ ] Sheepdog schedules a wake.
- [ ] Duration-only text without a rate-limit indicator does not schedule a wake.

### Manual panel path

- [ ] `/sheepdog` opens the interactive panel in TUI.
- [ ] Add wake previews the local wake time before save.
- [ ] Edit wake keeps manual override semantics.
- [ ] Clear selected and clear all require confirmation.

### Mismatch behavior

- [ ] Create a due wake for one scope, then switch to a different main model.
- [ ] No agent follow-up is sent.
- [ ] One short human notice appears for that entry.
- [ ] Wake remains pending until a matching model becomes active.

### Startup overdue behavior

- [ ] Start Pi after a pending wake time has already passed.
- [ ] No wake message is sent at startup.
- [ ] Wake becomes `expired` silently.

### Busy behavior

- [ ] Keep Pi busy when a matching wake becomes due.
- [ ] Footer shows `ready, waiting for idle` behavior.
- [ ] Follow-up waits until Pi is idle.

## Files and logs

- [ ] State path is `${agentDir}/.cache/sheepdog/state.json`.
- [ ] Debug log path is `${agentDir}/sheepdog/debug.log`.
- [ ] Logs redact secrets and avoid full provider payloads.
